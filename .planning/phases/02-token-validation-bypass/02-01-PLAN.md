      process.env.CLAUDE_CODE_ENTRYPOINT === 'sdk-ts' &&

## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | isVsCodeProxy() — env-var trust signal for bypass activation | accept | Three env vars must match simultaneously (ANTHROPIC_BASE_URL=http://localhost:*, CLAUDE_CODE_ENTRYPOINT=sdk-ts, CLAUDECODE=1). Setting all three requires existing process-level access; at that level the attacker already controls execution and gains nothing new from the bypass. |
| T-02-02 | Elevation of Privilege | getAnthropicApiKeyWithSource — approved-list gate bypass | accept | The approved list is a UX safeguard preventing repeated confirmation prompts, not a cryptographic authorization gate. An attacker with env var write access bypasses it trivially regardless of this change. No PII is stored; tokens are session-ephemeral and revoked when VS Code closes. |
| T-02-03 | Information Disclosure | ANTHROPIC_API_KEY (vscode-lm-* token) read earlier in the call chain | accept | The token is already present in process.env before openclaude starts — injected by VS Code. Reading it earlier in getAuthTokenSource creates no new exposure surface. Token has no value outside the VS Code session that issued it. |
</threat_model>
