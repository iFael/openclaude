## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → process.env | --provider vscode writes CLAUDE_CODE_ENTRYPOINT and CLAUDECODE into the current process environment |
| process.env → isVsCodeProxy() | isVsCodeProxy() trusts these env vars as a signal to skip token format validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | applyProviderFlag('vscode') | accept | Activation requires ownership of the process. A user who can pass --provider vscode can also set CLAUDE_CODE_ENTRYPOINT=sdk-ts manually — no new attack surface is created. Local process, same trust level. |
| T-03-02 | Elevation of Privilege | isVsCodeProxy() bypass of token format check | accept | The only "privilege" elevated is skipping the vscode-lm-* token rejection. ANTHROPIC_BASE_URL still routes to wherever the user set it. A user cannot escalate to another user's Copilot quota — the token is supplied by the user themselves. No ASVS L1 control is bypassed for a third party. |
</threat_model>
