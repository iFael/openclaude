<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Manual end-to-end test in VS Code integrated terminal</name>
  <read_first>
    - .planning/STATE.md — review Phase 1-3 decisions to know exactly what to look for
  </read_first>
  <what-built>
    All VS Code proxy integration changes from Phases 1-3 are compiled into the
    global `openclaude` binary (version 0.1.9). The binary should now:
      - Detect when running inside VS Code terminal (via ANTHROPIC_BASE_URL env var)
      - Print a proxy confirmation message at startup
      - Accept vscode-lm-* tokens without rejection
      - Skip OAuth and the approval dialog
      - Support --provider vscode as an explicit fallback flag
  </what-built>
  <how-to-verify>
    This test CANNOT be automated. GitHub Copilot Pro+ injects `ANTHROPIC_BASE_URL`
    and `ANTHROPIC_API_KEY` dynamically into VS Code's integrated terminal environment.
    These env vars are not present in a regular shell.

    Steps:

    1. Open VS Code with your GitHub Copilot Pro+ account signed in and Claude Code
       extension active.

    2. Open a new integrated terminal inside VS Code
       (Terminal > New Terminal, or Ctrl+` / Cmd+`).

    3. In that terminal, run:
         openclaude

    4. Look for the proxy detection message near the top of the startup output.
       Expected message (Phase 1 change):
         "Using VS Code Claude Code proxy"
       (or similar wording — the exact string is in src/main.tsx)

    5. Confirm you are NOT prompted for:
         - An Anthropic API key
         - An OAuth login flow
         - A token approval dialog
       (Phase 2 changes suppress all of these when proxy is detected)

    6. If the session opens normally, attempt one simple interaction (e.g., type
       a short message and press Enter). Confirm the session responds without a
       token format rejection error.

    7. Exit the session (Ctrl+C or /exit).

    Fallback (if ANTHROPIC_BASE_URL is not auto-injected):
       If you do not see the proxy message in step 4, try running:
         openclaude --provider vscode
       This uses the Phase 3 explicit flag. You will also need to set:
         export ANTHROPIC_BASE_URL=<the URL from VS Code's terminal env>
         export ANTHROPIC_API_KEY=<the token from VS Code's terminal env>
       Then re-run openclaude.
  </how-to-verify>
  <resume-signal>
    Type "approved" if the session started correctly with the proxy message and no
    token rejection errors.

    If something failed, describe what you saw (e.g., "got invalid API key error",
    "no proxy message appeared", "approval dialog showed up"). Include the error
    message if present.
  </resume-signal>
  <acceptance_criteria>

  </acceptance_criteria>
